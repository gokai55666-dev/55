## 7. Typical Bottlenecks to Watch Out For

| Bottleneck | What causes it | How to detect & fix |
|------------|----------------|----------------------|
| **GPU under‐utilization** | Small batch sizes, too many small requests | Monitor GPU utilization, if low but many requests are queued, adjust batching or combine requests. |
| **Cold starts** | Loading model, initializing frameworks | Warm pools, pre-load models, keep worker live. |
| **Data transfer overhead** | Moving inputs/outputs between CPU↔GPU frequently | Minimize copying, use pinned memory, process as much as possible on GPU. |
| **Memory limits / OOMs** | Model too big, too many concurrent inferences | Monitor memory, use pruning / quantization, restrict concurrency per GPU. |

## What the Medium Guide by Ishita Verma Covers: “PyTorch GPU Optimization: Step-by-Step Guide”

I went through the Medium article and pulled out actionable strategies, trade-offs, and best practices for optimizing PyTorch workflows on GPU.

### Key Optimization Techniques

1. **Ensure GPU Usage When Available**  
   - Use `torch.cuda.is_available()` to check and `.to(device)` to move models/tensors to GPU.  
   - Set `pin_memory=True` in DataLoader to speed CPU → GPU transfers. ([medium.com](https://medium.com/%40ishita.verma178/pytorch-gpu-optimization-step-by-step-guide-9dead5164ca2))

2. **Efficient Data Loading & Preprocessing**  
   - Use multiple workers in DataLoader (e.g. `num_workers > 0`) to parallelize data loading.  
   - Preprocess data ahead of training to avoid runtime bottlenecks. ([medium.com](https://medium.com/%40ishita.verma178/pytorch-gpu-optimization-step-by-step-guide-9dead5164ca2))

3. **Mixed Precision & Automatic/Manual Optimization**  
   - Use NVIDIA’s AMP (`torch.cuda.amp.autocast`) to run training in mixed precision (FP16/FP32), giving big speedups without large accuracy drops.  
   - Use loss scaling to avoid underflow issues. ([medium.com](https://medium.com/%40ishita.verma178/pytorch-gpu-optimization-step-by-step-guide-9dead5164ca2))

4. **Gradient Accumulation & Batch Size Handling**  
   - When GPU memory is limited, accumulate gradients over multiple smaller batches to simulate a larger batch.  
   - Adjust batch size to fill GPU memory without causing OOM errors. ([medium.com](https://medium.com/%40ishita.verma178/pytorch-gpu-optimization-step-by-step-guide-9dead5164ca2))

5. **Efficient Model Design & Memory Use**  
   - Use smaller or compact architectures where possible.  
   - Delete or detach unnecessary tensors to free memory. Use `with torch.no_grad()` for inference. ([medium.com](https://medium.com/%40ishita.verma178/pytorch-gpu-optimization-step-by-step-guide-9dead5164ca2))

6. **Optimizer & Learning Rate Strategies**  
   - Use optimizers like AdamW, or others more suited to your problem.  
   - Use learning rate schedulers to adjust rate in training. Warm-up phases help. ([medium.com](https://medium.com/%40ishita.verma178/pytorch-gpu-optimization-step-by-step-guide-9dead5164ca2))

7. **Profiling & Monitoring**  
   - Use CUDA memory summaries, PyTorch’s `torch.cuda.memory_summary()` or NVProf / Nsight to find and mitigate bottlenecks.  
   - Monitor GPU utilization via tools like `nvidia-smi`. ([medium.com](https://medium.com/%40ishita.verma178/pytorch-gpu-optimization-step-by-step-guide-9dead5164ca2))
