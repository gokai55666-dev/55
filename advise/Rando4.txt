### 1. Model Loader Module

**Goal:** Able to load different AI models (e.g. transformer‐based) and utilize GPU if available, else CPU.

**Files / Structure:**  
- `src/model_loader.py`

**Implementation steps:**

- Detect whether GPU is available. In Python, with PyTorch: `torch.cuda.is_available()`. If using TensorFlow or other framework, its equivalent.

- Based on detection, load model onto proper device:

  ```python
  device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
  model = ModelClass.from_pretrained(...).to(device)
  ```

- If model quantization or other optimizations are needed (e.g. load in 4-bit, half precision), ensure support: need appropriate libraries (bitsandbytes, accelerate, etc.).

**Dependencies & Versions:**

- PyTorch with CUDA support matching GPU driver. For example: if using NVIDIA GPU, CUDA toolkit version like 11.7 or 12.x. Must install `torch` built with matching CUDA: e.g. `torch==2.1.0+cu117`.

- For non-CUDA (AMD GPUs): ROCm builds (if supported) or fallback CPU-only.

- For quantization: bitsandbytes, llama.cpp, or other libs as needed; some require specific GPU architectures.

**Guide to Implement:**

1. Specify in `requirements.txt` something like:

   ```
   torch>=2.0.0; extra == "gpu"
   ```

   Or separate `requirements-gpu.txt`.

2. In code, encapsulate model loading:

   ```python
   from torch import cuda
   ...
   if cuda.is_available():
       device = "cuda"
   else:
       device = "cpu"
   ```

3. Allow user override via environment variable, e.g. `DEVICE=cpu` or `GPU_INDEX=0`.

4. Add unit test: assert that when `cuda.is_available()` is False, model loads without trying GPU (maybe via mocking).

**Potential Pitfalls:**

- Mismatch between installed PyTorch CUDA version and system drivers → fail.

- GPU memory insufficient → OOM. May need to support model sharding or smaller batch sizes.

- Bitsandbytes or similar libraries might fail to compile if GPU arch unsupported.

### 2. Inference / Pipeline Module

**Goal:** Process input (prompts), run inference, produce output, handle tokenization.

**Files / Structure:**  
- `src/inference.py`

**Implementation steps:**

- Load tokenizer (e.g. from HuggingFace). Tokenization must match model type.

- Inference function takes input text, encodes, sends through model, decodes output.

- If on GPU, ensure that encoding/decoding parts support GPU if possible; move tensors correctly.

- Handle streaming / non-streaming output if desired.

**Dependencies:**

- HuggingFace Transformers (or similar), version compatible with your model. If using pipeline abstraction, ensure GPU support enabled.

- Maybe accelerate or PEFT.

**Guides:**

1. In `requirements.txt` include:

   ```
   transformers>=4.x
   accelerate>=0.x
   ```

2. In inference module:

   ```python
   from transformers import AutoTokenizer, AutoModelForCausalLM
   ...

   tokenizer = AutoTokenizer.from_pretrained(...)
   model = AutoModelForCausalLM.from_pretrained(..., device_map="auto")
   ```

   Or manually move model to `device`.

3. If using large model that doesn't fit into one GPU, use `device_map="auto"` or sharding techniques.

4. Test both small prompt and large prompt. Measure latency and memory use.

## Sample Implementation Guide for GPU Install (NVIDIA + PyTorch)

1. **Check GPU and Driver**

   - Run `nvidia-smi` to check GPU model and driver version.

   - Ensure driver version supports the CUDA toolkit you plan to use.

2. **Install CUDA toolkit / cuDNN**

   - If using conda: `conda install cudatoolkit=11.7 cudnn -c pytorch -c nvidia`.

   - If using pip, ensure PyTorch wheel is for correct CUDA version.

3. **Install PyTorch with GPU support**

   ```bash
   pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu117
   ```

4. **Optional libs**

   - bitsandbytes: install via `pip install bitsandbytes`.

   - accelerate: `pip install accelerate`.

5. **Test GPU availability in Python**

   ```python
   import torch
   print(torch.cuda.is_available(), torch.cuda.get_device_name(0))
   ```

6. **Possible environment variables**

   - `CUDA_VISIBLE_DEVICES=0` to pick which GPU.

   - For mixed precision: `AMPERE` etc.
